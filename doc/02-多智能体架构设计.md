## 二、8个智能体角色定义

### Agent总览

| Agent | 角色名 | 核心职责 | 输入 | 输出 |
|-------|--------|---------|------|------|
| ProfileAgent | 学习分析师 | 对话抽取学生6维画像 | 用户聊天文本 | 6维画像JSON |
| DocAgent | 课程讲师 | 生成课程讲解文档 | 知识点+画像 | Markdown文档 |
| MindMapAgent | 知识架构师 | 生成知识点思维导图 | 课程大纲 | JSON树结构 |
| QuizAgent | 出题专家 | 生成练习题库 | 知识点+难度 | 结构化题目JSON |
| ReadingAgent | 学术推荐官 | 推荐拓展阅读材料 | 知识点+画像 | 阅读推荐列表 |
| CodeCaseAgent | 实战教练 | 生成代码实操案例 | 知识点+画像 | 代码教程(含可运行代码) |
| PathAgent | 学习规划师 | 规划个性化学习路径 | 画像+学习记录 | 学习路径JSON |
| TutorAgent | 辅导老师 | 即时答疑解惑 | 问题+知识上下文 | 文字+图解解答 |

### Agent详细设计

#### 1. ProfileAgent（学习分析师）

```
角色设定：你是一位资深的学习分析师，擅长通过对话了解学生的知识水平、学习风格和需求。
触发条件：每轮对话后自动触发
核心能力：
  - 从自然语言对话中抽取6维度特征
  - 增量更新画像（不覆盖已有信息，只更新变化项）
  - 识别隐含信息（如"我觉得公式太难"→认知风格偏实践型）
输出格式：JSON结构画像数据
```

#### 2. DocAgent（课程讲师）

```
角色设定：你是一位机器学习领域的资深讲师，善于用通俗易懂的方式讲解复杂概念。
触发条件：用户请求/路径推荐触发
核心能力：
  - 结合学生画像调整讲解深度（基础差→多举例；基础好→深入公式推导）
  - 结合知识库提供准确内容
  - 输出结构化Markdown（含标题/正文/公式/代码/小结）
```

#### 3. MindMapAgent（知识架构师）

```
角色设定：你是一位知识架构专家，擅长梳理知识体系和层次结构。
触发条件：用户请求/章节总结
核心能力：
  - 将课程内容组织为树形结构
  - 标注知识点之间的依赖关系
  - 输出JSON树结构（前端markmap渲染）
```

#### 4. QuizAgent（出题专家）

```
角色设定：你是一位经验丰富的出题专家，能根据知识点和难度精准出题。
触发条件：用户请求/阶段测试
核心能力：
  - 生成3种题型：选择题、填空题、编程题
  - 根据画像调整难度（易错点多→针对性出题）
  - 提供答案+详细解析
  - 输出结构化JSON
```

#### 5. ReadingAgent（学术推荐官）

```
角色设定：你是一位学术资源推荐专家，能精准匹配学生需求推荐学习资源。
触发条件：用户请求/路径推荐
核心能力：
  - 根据知识点推荐相关论文、博客、视频教程
  - 根据画像推荐难度匹配的资源
  - 提供推荐理由
```

#### 6. CodeCaseAgent（实战教练）

```
角色设定：你是一位机器学习实战教练，善于设计实操案例帮助学生理解算法。
触发条件：用户请求/路径推荐
核心能力：
  - 生成完整可运行的代码案例
  - 包含数据集说明、代码注释、运行结果
  - 根据画像选择合适的库（Scikit-learn/PyTorch）
```

#### 7. PathAgent（学习规划师）

```
角色设定：你是一位学习路径规划专家，能根据学生情况制定科学的学习计划。
触发条件：画像更新/用户请求
核心能力：
  - 结合知识点依赖图规划学习顺序
  - 根据画像定制路径（基础差→多基础；基础好→快进到高级）
  - 每个节点关联推荐资源
  - 支持动态调整（学完一步更新下一步）
```

#### 8. TutorAgent（辅导老师）

```
角色设定：你是一位耐心的机器学习辅导老师，能即时解答学生疑问。
触发条件：用户提问
核心能力：
  - 结合知识库提供准确解答
  - 多模态回答（文字解释+图解说明+代码示例）
  - 引导式教学（不直接给答案，先引导思考）
```

## 三、编排器设计（Orchestrator）

### 意图识别规则

| 用户意图关键词 | 调度Agent |
|--------------|----------|
| 首次对话/介绍自己/学习背景 | ProfileAgent |
| 讲解/解释/什么是/原理 | DocAgent |
| 思维导图/知识结构/梳理 | MindMapAgent |
| 练习/题目/测试/做题 | QuizAgent |
| 推荐/阅读/论文/资料 | ReadingAgent |
| 代码/实操/案例/项目 | CodeCaseAgent |
| 学习路径/规划/计划/怎么学 | PathAgent |
| 问题/疑问/为什么/怎么理解 | TutorAgent |

### 协作流程

#### 流程1：新用户首次使用

```
用户："我是大二计算机专业，线代基础还行，想学机器学习做推荐系统"

Step1: Orchestrator → ProfileAgent
  → 生成初始画像（知识基础：线代中等/编程基础待确认/ML零基础）
  → 学习目标：推荐系统方向

Step2: 画像触发 → PathAgent
  → 生成个性化学习路径（基础概念→经典算法→...→推荐系统实战）

Step3: 路径第一个节点触发 → 并行调用5个资源Agent
  → DocAgent("基础概念", 画像) → 课程文档
  → MindMapAgent("基础概念") → 思维导图
  → QuizAgent("基础概念", "入门") → 练习题
  → ReadingAgent("基础概念", 画像) → 推荐阅读
  → CodeCaseAgent("基础概念", 画像) → 代码案例

Step4: 资源打包 → 推送到前端资源中心
```

#### 流程2：用户学习过程中

```
用户："决策树的剪枝策略我不太理解"

Step1: Orchestrator识别 → TutorAgent
  → 查询知识库中"决策树剪枝"相关内容
  → 生成解答（文字+图解+代码示例）

Step2: 同时触发 → ProfileAgent（更新易错点维度）
  → 更新画像：weak_points += "决策树剪枝"

Step3: 画像变化触发 → PathAgent（动态调整路径）
  → 在当前节点增加"决策树剪枝"的补充资源
```
